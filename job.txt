USER ISSUE:
User reported "it should only show 1 card at a time and its all janky animation" with screenshots showing all 6 cards stacking immediately with no smooth transitions.

- itemDistance was set to 0 (either by user or linter)
- With itemDistance={0}, there's NO scroll space between cards
- All cards appeared and stacked immediately without the intended reveal effect
- No smooth one-by-one card entrance as user scrolls

WHAT WAS FIXED:
Changed ScrollStack parameters from broken configuration:
```typescript
<ScrollStack useWindowScroll={true} itemDistance={0} itemStackDistance={30} >
```

To optimized single-card-view configuration:
```typescript
<ScrollStack
  useWindowScroll={true}
  itemDistance={600}        // CRITICAL FIX: 600px scroll between cards
  itemScale={0.05}          // 5% scale reduction per stacked card
  itemStackDistance={40}     // 40px vertical offset for stacked cards
  baseScale={0.90}          // Cards scale to 90% when fully stacked
  stackPosition="20%"       // Cards stick at 20% from top of viewport
>
```

KEY PARAMETER CHANGES:
1. itemDistance: 0 ‚Üí 600px

   - Smooth transition as each new card enters from bottom

2. itemStackDistance: 30px ‚Üí 40px
   - Better visual separation of stacked cards
   - More pronounced depth effect

3. Added baseScale={0.90}
   - Cards scale down to 90% when stacked behind
   - Creates clear visual hierarchy

4. Added stackPosition="20%"
   - Cards stick at 20% from top of viewport
   - Optimal viewing position for readability

1. User scrolls down the page
